Design — Section 1 : Structure des fichiers

Organisation du projet Next.js. Chaque section Figma = un composant isolé.

Arborescence du projet

Racine du projet
📁 app/ — App Router Next.js
📄 layout.tsx — HTML racine, import fonts, ThemeProvider
📄 page.tsx — Home Page
📁 pricing/ → page.tsx
📁 about/ → page.tsx
📁 contact/ → page.tsx

📁 components/

📁 sections/ — HeroBasic, HeroActions, CardGrid, PanelImage...

Règle d'organisation des composants

ui/

Atomes réutilisables
Button, Input, Card, Badge, Avatar...

sections/

Blocs de page complets
HeroBasic, CardGridIcon, PanelImageContent, PageNewsletter...

layout/

Enveloppe globale
Header (avec nav + toggle), Footer

Animations Framer Motion — stratégie

Au scroll (sections)
  • fade-in + slide-up via useInView
  • Chaque section s'anime à l'entrée dans le viewport
Sur les éléments
  • Cards : scale + shadow au hover
  • Boutons CTA : animation légère
  • Transitions de pages via AnimatePresence
